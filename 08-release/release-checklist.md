*# Release Checklist*



*Purpose: Verify release readiness.*

*Used By: Development, QA.*



*## Checklist*



*\* Requirements completed*

*\* Acceptance criteria passed*

*\* Critical defects resolved*

*\* Documentation updated*

*\* Version assigned*

*\* Changelog updated*

*\* Release notes prepared*



*## Decision*



*Ready | Blocked*



*## AI Rule*



*Do not mark a release as ready if acceptance criteria are incomplete.*



*Prefer releasing small validated increments.*
